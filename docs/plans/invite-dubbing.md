# 초대 더빙 (배역 캐스팅 콜라보) 기획서

> 상태: 초안 · 작성 2026-06-23
> 한 줄: 내가 만든 만화의 배역을 친구에게 나눠 맡겨 **함께 더빙해 한 영상**을 완성하는 협업 기능. 앱의 핵심 성장·리텐션 엔진.

---

## 1. 배경 / 문제

- 현재 더빙은 **혼자** 모든 배역을 본인 목소리로 녹음한다(피드의 "나도 더빙"도 솔로 체험).
- 콘텐츠 소비는 되지만 **사람을 끌어오는 루프**가 없다 → 신규 유입·재방문 동력이 약함.
- 사용자 원안(비전 6번): "남이 만든 컷을 내가 **같이** 할 수도, 내가 작가가 될 수도" 의 "같이" 부분이 미구현.

## 2. 목표 / 성공 지표

- **바이럴 루프**: 만들기 → 친구 초대 → 친구 설치/실행·참여 → 친구도 제작·초대 (K-factor > 1 지향)
- **리텐션**: 초대받은 사람이 돌아와 자기 배역 완성, 완성 영상 공유
- 측정 지표(후속 product-analyst와 정의):
  - 초대 발송 수 / 초대 클릭률 / 클릭→설치·실행 전환율
  - 배역 완성률(초대받은 배역 중 녹음 완료 비율)
  - 콜라보 완성 영상 수 / 공유 수
  - 초대 참여자의 7일 재방문율, 자기 만화 제작 전환율

## 3. 핵심 개념 — "배역 캐스팅"

- 캐스팅 단위 = **캐릭터(화자)**. 지금도 컷마다 화자 입력 시 `Character`가 자동 생성됨 → 그게 곧 **배역**.
- 콜라보 1건 = **세션(CollabSession)**. 세션은 배역마다 **슬롯**을 가짐.
- 슬롯 담당자 상태: `내가 더빙` / `초대(비어있음)` / `친구가 맡음(claimed)` / `녹음 완료(recorded)`.
- 합본 영상 = 대사별로 **그 배역 담당자의 녹음**을 모아 합성.

## 4. 확정 결정사항 (추천대로 확정)

| # | 결정 | 선택 | 이유 |
|---|---|---|---|
| 1 | 초대 범위 | **비공개 1:1 친구 링크** (MVP) | 단순·신뢰. 공개 캐스팅(아무나 참여)은 Phase C |
| 2 | 합성 시점 | **모든 배역 녹음되면 호스트에게 "완성하기" 활성화 → 호스트가 트리거** (semi-auto) | 호스트 통제 + 조기 렌더 방지 |
| 3 | 빈 배역 처리 | **호스트가 직접 메꿀 수 있음**. 모든 배역이 차야 완성(부분 공개 X) | 결과물 완성도 보장 |
| 4 | 알림 | **MVP는 링크 기반 + 인앱/로컬 알림만. 원격 푸시(FCM)는 Phase C** | Firebase 외부 셋업 회피, 빠른 출시 |

## 5. 전체 플로우

```
[호스트]
1. 만들기 → 컷 + 대사 (현행 동일, 화자 = 배역 자동 추출)
2. [신규] 캐스팅 단계: 배역별 "내가 더빙" / "친구 초대" 지정
3. 발행(콜라보 세션 생성) → 초대 링크 생성
4. 공유 (카톡 등 · share_plus)
5. 내 배역 녹음 (있다면)

[친구]
6. 링크 탭 → 앱 열림(미설치면 스토어) → 만화 + 빈 배역 목록
7. "민수 맡기" → 민수 대사만 녹음 → 제출
8. (모든 배역 채워지면) 호스트에게 "완성 가능" 인앱 알림

[호스트]
9. "완성하기" → 배역별 녹음 모아 합본 렌더 → 완성 영상
10. 공유 → (루프 반복)
```

## 6. 데이터 모델 (신규/변경)

```
CollabSession
  id            text PK
  episodeId     text FK→Episode
  hostUserId    text FK→User
  shareCode     text unique          -- 딥링크용 짧은 코드
  status        enum(OPEN, READY, COMPLETE, CANCELLED)
  videoId       text? FK→RenderedVideo  -- 완성 영상
  createdAt / updatedAt

CollabRole
  id            text PK
  sessionId     text FK→CollabSession
  characterId   text FK→Character     -- 배역
  assignedUserId text?  FK→User       -- null = 비어있음(초대 대기)
  status        enum(OPEN, CLAIMED, RECORDED)
  @@unique(sessionId, characterId)

Recording (기존)
  -- 이미 (performanceId, dialogueId, userId)로 사용자별 저장됨.
  -- 콜라보는 "세션의 배역 담당자별 녹음"을 대사별로 선택해 합성.
  -- 합성 시: 각 Dialogue.characterId → CollabRole.assignedUserId → 그 유저의 Recording 사용.
```

- RLS: 세션/배역 조회는 `shareCode`로 접근 가능해야 하므로 **security definer RPC**(`collab_session_by_code`)로 노출(직접 테이블 RLS는 본인 것만 + 공개 RPC 병행). 배역 claim/녹음 제출은 인증 사용자 본인만.

## 7. 화면 (신규/변경)

- **캐스팅 화면**(발행 흐름에 삽입): 배역 리스트 → 각 배역 토글 "내가 / 초대". "발행 + 초대 링크 받기".
- **콜라보 관리 화면**(호스트): 배역별 상태(대기/맡음/완료), 초대 링크 재공유, 빈 배역 직접 더빙, "완성하기".
- **참여 화면**(친구, 딥링크 진입): 만화 미리보기 + 빈 배역 → "이 배역 맡기" → 퍼포머(배역 한정).
- **퍼포머 변경**: `roleCharacterId`가 있으면 그 배역 대사만 녹음하도록 `_lines` 필터.
- 피드/보관함에 "진행 중 콜라보" 섹션(내가 호스트거나 참여 중).

## 8. 기술 난관 (★ 주의)

1. **합본 렌더 (★ 핵심)**
   - 현재 온디바이스 렌더는 "내 녹음"만 사용. 콜라보는 대사마다 **다른 사용자 녹음**(Supabase Storage에 사용자별 저장됨)을 받아 합성해야 함.
   - 접근: **호스트가 완성 시 모든 배역 녹음을 다운로드 → 기존 온디바이스 합성 파이프라인 재사용**(대사별 오디오 소스만 담당자 것으로 교체). 서버 워커 렌더로 갈 수도 있으나 MVP는 호스트 온디바이스.
   - `local_render`/`Cloud.recordingMeta`가 단일 유저 가정 → 세션 기준으로 대사별 녹음 조회하도록 확장 필요.

2. **원격 푸시 (★ Phase C)**
   - 친구에게 "당신 배역을 기다려요" 푸시 = FCM 필요(현재 flutter_local_notifications 로컬만).
   - MVP는 푸시 없이 **공유 링크 + 인앱 알림**으로 동작. 넛지/리마인드 푸시는 FCM 붙이는 Phase C.

3. **딥링크**
   - `kr.co.pluck.dubbingo://collab/{code}` intent-filter 추가(OAuth 콜백용 intent-filter 이미 존재 → 패턴 재사용).
   - 미설치 사용자 대비 웹 랜딩(`/collab/{code}` → 스토어 유도)도 후속.

## 9. 단계 (Phasing)

- **A. 라벨 정직화** — "같이 더빙" → "나도 더빙" ✅ (완료)
- **B. 초대 더빙 MVP**
  - 캐스팅 화면 + CollabSession/CollabRole + shareCode RPC
  - 초대 링크 생성·공유, 딥링크 참여, 배역 한정 녹음·제출
  - 호스트 "완성하기" → 합본 렌더(호스트 온디바이스)
  - 인앱/로컬 알림(완성·참여), 빈 배역 호스트 직접 더빙
  - 제외: 원격 푸시, 공개 캐스팅
- **C. 성장 레이어**
  - FCM 원격 푸시(넛지·리마인드), 공개 "빈 배역" 콜라보 피드(아무나 참여), 미완성 리마인드, 웹 랜딩, 지표 대시보드

## 10. MVP 작업 분해 (예상)

1. DB: `CollabSession`/`CollabRole` 테이블 + RLS + `collab_session_by_code` RPC + Prisma 동기화
2. 발행 흐름에 캐스팅 단계 + `Cloud.createCollab(...)`(세션·배역·shareCode 생성)
3. 초대 링크 + share_plus 공유
4. AndroidManifest collab 딥링크 intent-filter + 앱 라우팅(코드 → 참여 화면)
5. 참여 화면 + `Cloud.claimRole` + 퍼포머 배역 필터 녹음·업로드
6. 콜라보 관리 화면(상태·재공유·빈배역 더빙·완성하기)
7. 합본 렌더 확장: 세션 대사별 담당자 녹음 수집 → 합성 → RenderedVideo + 세션 COMPLETE
8. 인앱/로컬 알림 + 보관함/피드 "진행 중 콜라보"
9. 실기기 E2E(호스트·참여자 2계정), 정리, 커밋

## 11. 리스크 / 열린 질문

- 참여자가 끝까지 안 하면? → 호스트가 직접 메꾸거나 리마인드(C). 만료/취소 정책 필요.
- 한 사람이 여러 배역? → 허용(혼자 일부 + 친구 일부). 모델상 자연 지원.
- 녹음 음질/길이 편차 합성 시 정렬 → 기존 세그먼트+concat 방식 유지, 대사별 길이 기준.
- 미설치 친구 전환 → 웹 랜딩 + 스토어 딥링크(C).
- 푸시 없는 MVP의 완성률 → 공유 메시지·인앱 알림으로 보완, 데이터 보고 C에서 FCM 투자 판단.
